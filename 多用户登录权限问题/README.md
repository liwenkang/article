## 问题描述

    当用户在浏览器中打开多个系统页面,并使用不同用户登录的时候,由于在本地的 localStorage 中只能保存一份用户登录数据,所以会产生覆盖的问题.

## 期待实现效果

    不同页面之间的用户信息应该独立保存,并在已登录状态下刷新页面不会丢失用户信息

## 解决方案

    之前的方案是把用户的相关信息,在登录成功后(由后端传值给我该用户的基本信息, 里面包含了该用户的相关权限信息),应该将其保存起来,但不能是 localStorage, 应该用 sessionStorage

## 涉及到的问题

    localStorage, sessionStorage, cookies 相关用法,以及用哪个来存储用户相关信息
    
    localStorage 
        保存的信息将会长期保留,但是如果把用户信息存在里面,当在页面中登录多个用户的时候,后面登录用户的数据信息就会覆盖掉之前的用户信息

    sessionStorage 
        当页面关闭时,存储的信息会清零,优点是,当我们在浏览器中,登录多个用户的时候,保存用户信息的sessionStorage 是分别保存的,不会发生覆盖的问题

    如果用户用系统管理员登录了,那么当他下次打开浏览器时,是否可以直接跳过登录页面?直接进入到需要的页面进行操作?为了安全考虑,不允许这样

    或者我们要求用户点击注销/直接关闭页面之后,自动销毁掉所有的用户信息,当用户下次直接点击某个非登录页面,应该直接跳转回登录的页面
